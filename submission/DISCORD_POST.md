Hydra MLX Troubleshooter

A local MLX model can repeat the same setup failure because a new session does
not know the device, compatibility rule, or earlier failure. This project uses
HydraDB Memory and Knowledge to preserve approved troubleshooting evidence
across sessions while model generation stays on the Mac.

The live demo starts with 0/3 required facts. It stores a measured M4 Pro and
24 GB device profile, an MLX VLM compatibility runbook, and an inferred Memory
from a previous KV cache load failure. A fresh HydraDB client then recalls 3/3
facts. The local 27 billion parameter MLX model uses them to reject the unchanged
configuration. A policy gate blocks credentials before persistence.
